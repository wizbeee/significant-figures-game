# 🚀 학교 노트북 Claude Code 시작 프롬프트

> 학교 노트북에서 Claude Code를 처음 켰을 때 **아래 프롬프트 전체를 복사해 붙여넣기** 하세요.
> Claude Code가 환경 점검부터 서버 실행까지 단계별로 안내합니다.

---

## 📋 복사해서 붙여넣을 프롬프트 (최초 1회 셋업용)

```
유효숫자 마스터 게임을 학교 노트북에서 서버로 운영하려고 해. 처음부터 끝까지 같이 셋업해줘.

# 내 상황
- 충남삼성고 화학교사 홍권능
- 학교 Windows 노트북 (단일 PC 운영, Google Drive 동기화 X)
- Node.js / Git 설치 여부 모름
- 학생들은 같은 학교 Wi-Fi 에서 이 노트북 IP로 접속할 예정
- 첫 시즌은 2026-05-12 프리시즌 시작 예정
- GitHub: https://github.com/wizbeee/significant-figures-game (Public)

# 해줄 것 (이 순서대로, 단계마다 결과 확인 후 다음으로)

## 1단계 — 환경 점검
- PowerShell에서 `node --version` 실행 → v18 이상이어야 함. 안 깔려있으면 https://nodejs.org/ LTS 설치 안내 (설치 후 PowerShell 재시작 필요)
- `git --version` 확인 → 없으면 https://git-scm.com/download/win 안내
- 둘 다 OK 되면 다음 단계로

## 2단계 — 프로젝트 클론
- `C:\sigfig` 폴더에 클론
  ```
  cd C:\
  git clone https://github.com/wizbeee/significant-figures-game.git sigfig
  cd sigfig
  ```
- 클론 후 `학교노트북_셋업.md` 와 `HANDOFF_NEXT.md` 를 먼저 읽고 현재 상태 파악
- 그 두 문서가 이 셋업의 정식 안내서. 그걸 따르면서 진행해줘

## 3단계 — 방화벽 허용 (포트 8093)
- 관리자 PowerShell 실행 가능한지 확인
- 가능하면: `New-NetFirewallRule -DisplayName "sigfig-game" -Direction Inbound -Protocol TCP -LocalPort 8093 -Action Allow`
- 불가능하면: 첫 서버 실행 시 뜨는 Windows Defender 팝업에서 "개인 네트워크" 체크 + 액세스 허용 안내

## 4단계 — 첫 실행 (스모크 테스트)
- `node server.js` 실행
- "🚀 sigfig-game ready" 메시지와 LAN IP 출력 확인
- localhost:8093 으로 브라우저 열어서 홈 화면 뜨는지 확인 (가능하면)
- `Ctrl+C` 로 정상 종료 → "graceful shutdown" 메시지 확인

## 5단계 — 운영용 실행 (토큰 48시간)
- 다음 명령으로 시작:
  ```
  $env:STUDENT_TOKEN_TTL_HOURS=48
  node server.js
  ```
- LAN IP 다시 확인하고 학생 안내용 URL 정리해줘:
  - 학생: http://<LAN_IP>:8093/
  - 교사: http://<LAN_IP>:8093/teacher.html
  - 점수판: http://<LAN_IP>:8093/leaderboard.html
- `ipconfig` 로 IP 직접 확인하는 방법도 알려줘

## 6단계 — 첫 시즌 시작 준비 (선택)
- 교사 페이지(/teacher.html) 첫 로그인 절차 안내
  - 교실 코드 (예: 2학년화학) + 비밀번호 설정 → 로그인/교실 만들기
- 학생 CSV 일괄 등록 형식 안내 (반·번호·이름)
- 시즌 설정 권장값:
  - 시즌 이름: "2026 프리시즌 1"
  - 활성 시간 윈도우: 🍱🍽 점심+저녁
  - 시작: 즉시 / 종료: 2026-05-18 23:59
- 학생들에게 IP+코드 안내 방법 (QR 코드 출력 등)
```
