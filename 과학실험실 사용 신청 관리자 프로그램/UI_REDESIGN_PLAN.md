# 과학실험실 관리자 프로그램 UI/UX 재설계 계획서

> 작성일: 2026-04-08
> 대상: 과학 실험·실습실 사용 신청 관리 시스템 (관리자)
> 목표: 데스크탑 / 태블릿 / 스마트폰 — 3가지 레이아웃, 동일 기능

---

## 1. 현재 구조 요약

### 1.1 페이지 구성
| 페이지 | 파일 | 핵심 기능 |
|--------|------|-----------|
| 로그인 | `entry.html` | 교사ID/비밀번호 로그인, 자동로그인, 다크모드 |
| 대시보드 | `admin.html` | 통계카드, 필터/정렬, 신청 테이블, 편집모드, 모달 |
| 시약 목록 | `chemical_list.html` | 기간별 시약 조회, 3단계 정렬, 그룹화, 인쇄/엑셀 |
| 학생 명단 | `student_list.html` | 학생 조회, 5가지 보기모드, 출석체크, 인쇄/엑셀 |
| 임장 일정 | `teacher_schedule.html` | 교사별 일정, 표/달력 보기, 다중선택, 수정 모달 |

### 1.2 현재 문제점
1. **반응형이지만 레이아웃이 동일** — 모바일에서 데스크탑 UI를 축소한 수준
2. **컨트롤 영역 과밀** — 필터/정렬/버튼이 화면 절반 이상 차지
3. **네비게이션 불편** — 페이지 간 이동이 직관적이지 않음
4. **모바일 터치 최적화 부족** — 작은 버튼, 복잡한 드롭다운
5. **코드 1벌로 3가지 화면 대응** — CSS 분기 한계

## 2. 재설계 원칙

### 2.1 플랫폼 분리 전략
```
데스크탑 (Desktop)     → 브라우저 전용 HTML/CSS/JS
                         로그인 화면 포함
모바일   (Mobile App)  → Android WebView 전용 HTML/CSS/JS
                         ├─ 스마트폰 (≤ 480px)  반응형
                         └─ 태블릿  (481~1024px) 반응형
                         ⚠️ 로그인 화면 없음 (앱 자체 인증 또는 자동 로그인)
                         ⚠️ Pull-to-refresh(아래로 당겨 새로고침) 비활성화
```

### 2.4 모바일 전용 제약사항
1. **로그인 화면 제거**
   - Android 앱에서 로그인 정보를 SharedPreferences에 저장
   - WebView 로드 시 자동으로 인증 파라미터 전달 (`?auth=token`)
   - 또는 앱 최초 실행 시 네이티브 로그인 화면 → 이후 자동 입장
   - `entry.html`을 거치지 않고 바로 `admin.html`로 진입

2. **Pull-to-refresh 비활성화**
   - WebView의 `SwipeRefreshLayout` 제거 또는 비활성화
   - 스크롤 중 의도치 않은 새로고침 방지 (정보 조회 방해 제거)
   - 새로고침이 필요한 경우 → 앱 내 별도 새로고침 버튼 제공
   - Android 코드: `swipeRefreshLayout.setEnabled(false)`

### 2.2 공유 요소
- **백엔드** (Code.gs) — 완전히 동일, 수정 없음
- **비즈니스 로직 JS** — 공통 모듈로 추출 (data fetching, filtering, sorting)
- **디자인 토큰** — 색상, 폰트, 간격 등 CSS 변수 공유

### 2.3 분리 요소
- **레이아웃 HTML** — 데스크탑/모바일 각각 별도
- **스타일 CSS** — 플랫폼별 최적화
- **인터랙션 JS** — 터치 vs 마우스 이벤트 처리

function doGet(e) {
  const view = e.parameter.view || 'entry';
  const platform = e.parameter.platform || 'desktop';

  if (view === 'entry') {
    return HtmlService.createTemplateFromFile('entry')
      .evaluate()
      .setTitle('로그인')
      .addMetaTag('viewport', 'width=device-width, initial-scale=1');
  }

  // 플랫폼에 따라 다른 HTML 제공
  const prefix = platform === 'mobile' ? 'mobile/' : 'desktop/';

  switch (view) {
    case 'admin':
      return serve(prefix + 'admin');
    case 'chemical_list':
      return serve(prefix + 'chemical_list');
    case 'student_list':
      return serve(prefix + 'student_list');
    case 'teacher_schedule':
      return serve(prefix + 'teacher_schedule');
  }
}

## 9. 데스크탑 vs 모바일 기능 대비표

| 기능 | 데스크탑 | 모바일 (폰/태블릿) |
|------|----------|---------------------|
| 로그인 | entry.html (교사ID/비밀번호) | ❌ 없음 (앱 자동 인증) |
| 새로고침 | 브라우저 기본 (F5) | ❌ Pull-to-refresh 비활성, 앱 내 버튼 제공 |
| 네비게이션 | 좌측 사이드바 | 하단 탭 바 + 햄버거 메뉴 |
| 통계 카드 | 수평 5열 그리드 | 수평 스크롤 캐러셀 |
| 필터 | 1줄 필터 바 + 접이식 고급 | 빠른 기간 버튼 + 접이식 |
| 데이터 표시 | 테이블 (정렬 가능 헤더) | 카드 리스트 |
| 편집 모드 | 인라인 테이블 편집 | 상세 화면 내 편집 |
| 일괄 작업 | 체크박스 + 벌크 바 | 다중 선택 모드 + 하단 액션 |
| 상세 보기 | 모달 (720px) | 풀스크린 페이지 |
| 정렬 | 4단계 다중 정렬 | 1~2단계 간소화 정렬 |
| 달력 | 큰 월간 캘린더 (셀에 일정 표시) | 점 표시 달력 + 일별 카드 |
| 인쇄 | 별도 인쇄 창 | 동일 (시스템 인쇄) |
| 엑셀 내보내기 | CSV 다운로드 | CSV 다운로드 (동일) |
| 다크모드 | 지원 | 지원 |
| 로그아웃 | 사이드바 하단 | 햄버거 메뉴 내 |
