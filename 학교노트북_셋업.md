# 🏫 학교 노트북 셋업 가이드 (한 페이지)

> 이 문서를 따라 한 번만 셋업하면, 매일 점심·저녁 시간에 명령 한 줄로 서버를 켜고 끌 수 있습니다.

**전제**:
- 학교 Windows 노트북 1대를 서버 PC로 사용
- 학생·교사·점수판 모두 같은 학교 Wi-Fi에서 그 노트북 IP로 접속
- Google Drive 동기화 사용 X (단일 PC 운영이라 불필요)
- 데이터는 노트북 로컬 디스크에 누적 (자동 백업 14일치 + 학기말 수동 백업)

---

## 🪜 최초 1회 셋업 (대략 15분)

### Step 1. Node.js 18+ 설치 (5분)

1. https://nodejs.org/ 접속
2. **LTS 버전** (왼쪽 큰 버튼) 다운로드 → 설치 (모든 옵션 기본값으로 Next)
3. 확인:
   ```powershell
   node --version
   # v18.x.x 또는 v20.x.x 등 → OK
   ```

### Step 2. Git 설치 (3분)

1. https://git-scm.com/download/win 접속 → 자동 다운로드
2. 설치 (모든 옵션 기본값으로 Next)
3. 확인:
   ```powershell
   git --version
   # git version 2.x.x → OK
   ```

### Step 3. 프로젝트 클론 (1분)

PowerShell 또는 명령 프롬프트 열기:

```powershell
# 원하는 폴더로 이동 (예: C 드라이브 루트)
cd C:\

# 프로젝트 폴더 생성하며 클론
git clone https://github.com/wizbeee/significant-figures-game.git sigfig

# 들어가기
cd sigfig
```

> 💡 폴더 위치는 자유. `D:\` 혹은 `바탕화면` 등 어디든 OK. 단 한국어 폴더명/공백은 피하는 게 안전.

### Step 4. 방화벽 허용 (1분)

처음 `node server.js` 실행 시 **Windows Defender 방화벽** 팝업이 뜨면 → "**개인 네트워크**" 체크 → **액세스 허용** 클릭.

(팝업이 안 뜨면 미리 수동으로 허용. 관리자 PowerShell에서:)
```powershell
New-NetFirewallRule -DisplayName "sigfig-game" -Direction Inbound -Protocol TCP -LocalPort 8093 -Action Allow
```

### Step 5. 첫 실행 (5초)

```powershell
node server.js
```

화면에 이런 메시지가 나오면 성공:

```
🚀 sigfig-game ready  http://localhost:8093
   LAN  http://10.1.x.x:8093
```

→ **`LAN` 줄의 IP**를 학생들에게 안내하면 됩니다.

종료는 `Ctrl + C` (창을 X로 닫지 말 것).

## 🌅 매일 운영 (점심·저녁 30초)

### 켤 때

```powershell
cd C:\sigfig\significant-figures-game

# 토큰 48시간 (점심→저녁 사이 학생 재로그인 방지)
```

학생 안내: `http://10.1.x.x:8093/` (그날 LAN IP로)

PowerShell 창에서 `Ctrl + C` →
```
🛑 graceful shutdown — 진행 중 게임 finalize 완료
```
메시지 뜨면 노트북 닫기 OK.

### 가끔 (선택) — 최신 코드 받기

```powershell
cd C:\sigfig\significant-figures-game
git pull origin main
```

## 📋 사용자 URL 정리

| 누가 | URL | 용도 |
|---|---|---|
| **학생** | `http://10.1.x.x:8093/` | 게임 플레이 |
| **교사** | `http://10.1.x.x:8093/teacher.html` | 시즌·학생·방·분석 관리 |
| **점수판** | `http://10.1.x.x:8093/leaderboard.html` | 교실 TV에 띄워놓기 |

> `10.1.x.x` 는 **그날 노트북 IP**. 매번 `ipconfig` 로 확인 (Wi-Fi 재연결 시 바뀔 수 있음).

```powershell
ipconfig | Select-String "IPv4"
```

## 🎓 첫 운영 흐름 (5/12 프리시즌 시작 예시)

1. **5/12 (월) 12:30 직전** — 노트북 켜고 서버 시작
2. **교사 페이지 접속** (`/teacher.html`)
   - 첫 로그인: 교실 코드 입력 (예: `2학년화학`) + 비밀번호 설정 → 로그인 / 교실 만들기
3. **학생 등록** — 👥 학생 탭 → CSV 일괄 등록 (반·번호·이름)
4. **시즌 시작** — 🏆 시즌 탭
   - 시즌 이름: `2026 프리시즌 1`
   - 활성 시간 윈도우: **🍱🍽 점심+저녁** 한 번에
   - 시작: 즉시 / 종료: 5/18 23:59
   - 🚀 시즌 시작
5. **학생 안내** — 화면에 IP + 교실 코드 띄워주기 (또는 🔗 공유 → QR 출력 → 칠판에 붙이기)
6. **점심 시간 운영** — 학생들이 입장해서 게임
7. **13:30** — `Ctrl+C` 로 정상 종료
8. **18:30** — 다시 켜고 운영 (토큰 48시간이라 학생 자동 로그인 유지)
9. **5/18 23:59** — 시즌 자동 종료 + 자동 랭크 부여 + 챔피언 발표

## 🆘 문제 해결

| 증상 | 해결 |
|---|---|
| 학생이 "사이트에 연결할 수 없음" | ① 같은 Wi-Fi 인지 확인 ② 방화벽 허용했는지 ③ IP 다시 확인 (`ipconfig`) |
| 서버 시작 시 `EADDRINUSE :8093` | 다른 곳에서 이미 실행 중. 다른 PowerShell 창 종료 또는 `$env:PORT=8094` 로 다른 포트 |
| 노트북 슬립 들어가서 학생 끊김 | 전원 옵션 → "절전 안 함" 또는 "디스플레이 끄기만" 으로 변경 |
| 점심에 로그인했는데 저녁에 다시 로그인 요구 | `$env:STUDENT_TOKEN_TTL_HOURS=48` 환경변수 설정했는지 확인 |
| 노트북 재부팅 후 IP 바뀜 | 학교 IT에 노트북 MAC 주소 고정 IP 할당 요청 (선택) |
| `git pull` 충돌 | 로컬에서 코드 수정 X. 코드 변경은 다른 PC에서 push → 학교 노트북에서 pull 만 |

## 💾 데이터 안전성

자동 보호 3중:
1. **atomic write** — 쓰는 도중 전원 끊겨도 손상 없음
2. **자동 일별 백업** — `data/backups/YYYY-MM-DD/` 14일치 자동 생성·로테이션
3. **graceful shutdown** — `Ctrl+C` 시 진행 중 게임 finalize 후 저장

학기말 한 번:
- 교사 페이지 → 💾 데이터 탭 → **🗄 전체 백업 (JSON)** → USB나 클라우드에 보관
